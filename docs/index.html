homeport

A self-hosted personal hub. Aggregates independent satellite services into one dashboard — no lock-in, no domain logic.

Architecture

Homeport is a pure aggregation layer. The hub reads from independent satellite services and renders widget cards. Each satellite is a standalone Docker container — usable without homeport.

hub  →  fetches GET /widget from each satellite, renders a card grid

infra  →  Docker monitoring: container status, CPU, RAM, disk

inventory  →  equipment and project tracker with shopping list

third-party  →  Vikunja (tasks), Actual Budget (finances), Quartz (notes) — link cards, no code needed

Features

hub + satellites
Add a new domain by deploying a container and adding one line to satellites.json. The hub never changes.
widget protocol
Each satellite exposes GET /widget — a standard JSON response with status, summary, and metrics.
infrastructure satellite
Live container cards, system metrics, stop/restart/start actions, and Dozzle log links.
inventory satellite
Track equipment by category, quantity, and location. Assign items to projects. Shopping list for low-stock items.
shared UI
All satellites use @homeport/ui — a shared React component library with a consistent dark theme.
no vendor lock-in
Third-party tools (Vikunja, Actual Budget) are first-class satellites — just link cards, no integration code.

Quick start

# clone and configure
git clone https://github.com/sensokame/homeport
cd homeport

# edit satellites.json to match your setup
cp apps/hub/satellites.json.example apps/hub/satellites.json

# start the hub
docker compose -f apps/hub/docker-compose.yml up -d

# start the infra satellite
docker compose -f apps/infra/docker-compose.yml up -d
